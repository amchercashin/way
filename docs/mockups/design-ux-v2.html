Решения по дизайну

Консистентность stat-блоков
На всех трёх уровнях (Главный → Категория → Актив) одинаковая четвёрка: Доход/мес, Стоимость, Доходность, Доля портфеля. На уровне актива «доля портфеля» = доля в общем портфеле (2.9%), а не в категории.
CAGR на графиках
На каждом уровне — столбчатая диаграмма истории выплат с CAGR в правом верхнем углу. Показывается только при наличии 2+ годовых периодов. Визуальная симметрия от общего к частному.
Ожидаемый дивиденд
Блок с зелёной рамкой на экране актива. Три даты: отсечка, выплата, зачисление. Данные из MOEX (если объявлены) или экстраполяция из прошлой выплаты (с пометкой «ожид.»). Показывается только для биржевых активов.
Обновить vs Добавить — ключевое UX-решение
Два режима импорта с принципиально разной семантикой:

⟳ Обновить = «вот мой полный портфель на сегодня». Система сравнивает с текущим состоянием, показывает diff (что изменилось, что новое). Пользователь подтверждает.

+ Добавить = «вот новые сделки». Они добавляются поверх существующих позиций (кол-во увеличивается). Полезно когда пользователь хочет добавить только свежие покупки.

По умолчанию активен «Обновить» — это безопаснее и понятнее для большинства.
AI-импорт: Markdown как формат
Markdown-таблица вместо CSV/JSON: человекочитаемая (легко проверить глазами), AI отлично генерирует, парсинг тривиален. Пользователь копирует промт → отправляет с файлом отчёта в любой AI → вставляет результат. Работает с любым брокером без нашей интеграции.
