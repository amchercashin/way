# Онбординг — Спецификация v2

**Дата:** 2026-03-26
**Статус:** Approved
**Предыдущая версия:** `2026-03-25-onboarding-design.md` (откачена — визуальные отклонения при реализации)

## Обзор

Двухфазный онбординг для HeroIncome: короткий гайд-тур при первом запуске (фаза 1) + контекстные подсказки при первом посещении страниц (фаза 2). Кастомная реализация на CSS overlay + Floating UI, без внешних библиотек.

**Ключевое отличие от v1:** spotlight реализован как CSS `radial-gradient` («фонарик»), стрелки убраны полностью. Это упрощает архитектуру и устраняет проблемы предыдущей реализации (овальные вырезы, z-index конфликты, сложные SVG path).

## Фаза 1: First Launch Tour

Запускается сразу после splash-экрана (HI! → HeroIncome). 7 шагов, ~40 секунд. Кнопка «Пропустить» на каждом шаге.

### Шаг 1 — Мокап заполненного приложения

Полноэкранный оверлей со статичным скриншотом главной страницы с примерными данными. Поверх — полупрозрачный затемнённый слой с текстом:
- «Так выглядит приложение с данными»
- Внизу: «нажмите для продолжения»

**Важно:** скриншот — React-компонент (MockMainPage), pixel-perfect копия реального UI. При изменениях main page — обновлять мокап.

Переход: тап в любое место.

### Шаг 2 — Реальное пустое состояние

Мокап уходит (fade-out ~300ms), видна реальная пустая главная страница. Тултип по центру экрана:
- «А пока здесь пусто. Давайте это исправим!»

Переход: тап в любое место.

### Шаг 3 — Указатель на меню

Spotlight-фонарик подсвечивает кнопку ☰. Тултип рядом (позиционирование Floating UI):
- «Нажмите для доступа к меню»

Переход: пользователь реально нажимает ☰ — меню открывается.

### Шаг 4 — «Данные» в меню

Меню открыто. Spotlight подсвечивает пункт «Данные». Тултип:
- «Управление портфелем — здесь»

Переход: пользователь нажимает «Данные» → навигация на /data.

### Шаг 5 — Страница Данные

На странице /data появляется тултип (без spotlight — на новой странице нет конкретного target-элемента):
- «Импортируйте брокерский отчёт или добавьте активы вручную»
- Подпись: *Для биржевых активов выплаты подтянутся автоматически*

Переход: тап в любое место.

### Шаг 6 — Добавление тестового актива

Карточка-предложение по центру:
- «Добавим 1000 акций Мосбиржи (MOEX) для примера?»
- *Удалить можно в любой момент*
- Кнопка **[Добавить]**

По нажатию кнопки:
1. Программно создаётся актив: `{ ticker: 'MOEX', name: 'Мосбиржа', type: 'Акции', dataSource: 'manual' }`
2. Если нет ни одного аккаунта — создаётся: `{ name: 'Основной' }`
3. Создаётся holding: `{ accountId, assetId, quantity: 1000, quantitySource: 'manual' }`
4. Запускается `syncSingleAsset(assetId)` напрямую (не через `syncAsset()` контекста — тот может быть заблокирован начальной синхронизацией)
5. Переход на главную страницу через `withViewTransition(() => navigate('/'))`
6. Пауза ~2 сек для показа рассчитанного дохода

### Шаг 7 — Удаление и чистый старт

Карточка поверх главной с рассчитанным доходом:
- «Вот как выглядит расчёт дохода. Удаляем тестовый актив — начните со своих данных!»
- Кнопка **[Удалить и начать]**

По нажатию:
1. `deleteAsset(id)` — каскадно удаляет актив, holdings, paymentHistory (существующая логика в `use-assets.ts`)
2. Доп. логика: проверить, был ли аккаунт создан во время онбординга (запомнить id при создании на шаге 6) и пуст ли он (0 holdings) — если да, удалить
3. Переход на пустую главную через `withViewTransition(() => navigate('/'))`
4. `localStorage.setItem('hi-onboarding-done', '1')` (значение `'1'` — достаточно; если понадобится ре-триггер при мажорных обновлениях, перейти на `__APP_VERSION__`)

### Кнопка «Пропустить»

Видна на каждом шаге (мелкий текст, цвет `--hi-ash`). По нажатию:
- Если тестовый актив уже создан (шаги 6-7) — удалить его
- Закрыть drawer если открыт
- Навигировать на `/` если не на главной
- Установить `hi-onboarding-done = 1`
- Завершить тур

## Фаза 2: Контекстные подсказки по страницам

Появляются автоматически при первом посещении каждой страницы. Spotlight-фонарик + тултип. Тап в любое место закрывает.

**Условие:** PageTip показываются только если `hi-onboarding-done` установлен (фаза 1 завершена или пропущена).

### Страница «Выплаты» (/payments)

- **Target:** область списка выплат
- **Текст:** «История выплат. Для биржевых активов подтягивается с MOEX, но можно добавить и вручную»
- **Ключ:** `hi-tip-payments`

### Страница категории (/category/:type)

- **Target:** первая строка актива
- **Текст:** «Доход и доходность — по каждому активу отдельно. Нажмите для подробностей»
- **Ключ:** `hi-tip-category`

### Страница актива (/asset/:id)

- **Target:** поле «Выплата на шт. / год» с бейджем источника
- **Текст:** «Для биржевых активов годовой доход рассчитан по последним выплатам. Но можно указать и вручную — полезно если знаете точнее»
- **Ключ:** `hi-tip-asset`

### Сброс из настроек

В SettingsPage добавляется кнопка «Сбросить подсказки». Удаляет все 4 ключа localStorage. Если `hi-onboarding-done` сброшен, но в БД есть активы — фаза 1 запускается без шагов 6-7 (добавление/удаление тестового актива пропускается).

## Визуальный дизайн

### Spotlight-фонарик

- `position: fixed`, `inset: 0`, `z-index: 9000`
- Реализация: CSS `radial-gradient` на div-элементе
- Эффект «фонарика»: мягкий плавный переход от прозрачности в центре к затемнению по краям
- Два слоя:
  1. **Тень:** `radial-gradient(ellipse <rx> <ry> at <cx> <cy>, transparent 0%, transparent 20%, rgba(0,0,0,0.15) 40%, rgba(0,0,0,0.55) 60%, rgba(0,0,0,0.82) 80%, rgba(0,0,0,0.88) 100%)`
  2. **Тёплый свет:** `radial-gradient(ellipse <rx2> <ry2> at <cx> <cy>, rgba(255,248,230,0.07) 0%, rgba(255,240,210,0.03) 40%, transparent 65%)`
- Без рамок, контуров и SVG filter — только gradient
- Размер пятна подбирается под target: padding ~15-20px от getBoundingClientRect()
- Параметры (opacity, размер, теплота) доводятся визуально при реализации
- **Шаги 1-2, 5-7:** `pointer-events: auto` на оверлее — перехватывает тап для перехода к следующему шагу
- **Шаги 3-4:** `pointer-events: none` на оверлее — клики проходят к реальным элементам (☰, пункт меню «Данные»)

### Тултип

- Фон: SVG path органической формы (неровные контуры), fill `--hi-stone` (#1a1815)
- Рамка: `stroke: rgba(200,180,140,0.18)`, stroke-width 1
- Текст: `--hi-text` (#b0a898), font-family DM Sans, `--hi-text-body` (12-14px)
- Счётчик шагов: `--hi-muted` (#4a4540), font-family IBM Plex Mono, `--hi-text-micro`
- Позиционирование: Floating UI (`computePosition` + `offset`/`flip`/`shift` middleware)
- Вход: fade + scale(0.95→1.0), 200ms ease-out

### Подпись внизу

- Текст: «нажмите для продолжения»
- Цвет: `--hi-ash` (#5a5548)
- Шрифт: IBM Plex Mono, `--hi-text-micro`
- Позиция: `position: absolute; bottom: 24px; left: 50%; transform: translateX(-50%)`

### Анимация между шагами

- Spotlight плавно перемещается: CSS transition ~300ms ease-in-out на gradient position/size
- Тултип: fade-out → fade-in с новой позицией
- При смене страницы (шаги 4→5): мгновенное появление без морфинга

## Edge cases

### Навигация назад посреди тура

Если пользователь нажимает «назад» или навигирует вручную во время фазы 1 — тур сбрасывается: закрывается оверлей, тестовый актив удаляется (если создан), `hi-onboarding-done` устанавливается. FirstLaunchTour слушает route changes и при несовпадении ожидаемого роута — завершает тур.

### Sync failure на шаге 6

Если MOEX API недоступен — тестовый актив создаётся, но доход будет 0₽. На шаге 7 показать текст: «Данные с биржи загрузятся позже. Пока доход не рассчитан — это нормально.» Тур продолжается штатно.

### PageTip на пустой странице

Если target-элемент не найден в DOM (ref.current === null) — PageTip не рендерится. Подсказка покажется при следующем посещении, когда элемент будет в DOM.

### Drawer на шагах 3-4

FirstLaunchTour должен иметь возможность открывать/закрывать drawer. Решение: OnboardingContext в App.tsx предоставляет `drawerOpen: boolean` и `setDrawerOpen: (v: boolean) => void`. AppShell при активном онбординге читает `drawerOpen` из контекста и передаёт в `<Sheet open={drawerOpen} onOpenChange={setDrawerOpen}>`. Когда онбординг не активен — AppShell использует локальный useState как сейчас. На шаге 3 после клика на ☰ — tour вызывает `setDrawerOpen(true)` → шаг 4. На шаге 4 после клика на «Данные» — `setDrawerOpen(false)` + `withViewTransition(() => navigate('/data'))` → шаг 5.

### Floating UI пакет

Проверить при реализации: если `@floating-ui/react-dom` (зависимость Radix) достаточен — использовать его. Если нужен `@floating-ui/react` — установить. Функциональность нужна минимальная: `computePosition` + `offset`/`flip`/`shift`.

## Архитектура

### Компоненты

```
src/components/onboarding/
  CoachOverlay.tsx      — CSS div с radial-gradient (затемнение + spotlight-фонарик)
  CoachTooltip.tsx      — Тултип (органическая SVG-форма + текст + Floating UI)
  FirstLaunchTour.tsx   — Фаза 1: стейт-машина 7 шагов
  MockMainPage.tsx      — Статичный мокап заполненной главной страницы (шаг 1)
  PageTip.tsx           — Фаза 2: обёртка для контекстных подсказок
```

### CoachOverlay

Принимает `targetRect: DOMRect | null`. Рендерит `position: fixed` div с CSS `radial-gradient`, центрированным на target. Два слоя: тень + тёплый свет. При `targetRect === null` — полноэкранное равномерное затемнение (шаги 1-2, 5-7). `ResizeObserver` / scroll listener для пересчёта позиции.

### CoachTooltip

Принимает `targetRef`, `text`, `stepIndex`, `totalSteps`. Использует Floating UI для позиционирования. Рендерит SVG background path органической формы. Показывает текст + счётчик.

### FirstLaunchTour

Стейт-машина: `step: 0|1|2|3|4|5|6|7` (0 = не начат, после 7 = done).

Логика:
1. Проверяет `localStorage.getItem('hi-onboarding-done')` — если есть, не рендерится
2. Шаги 1-2: полноэкранные карточки (targetRect = null)
3. Шаги 3-4: spotlight на реальных элементах (☰, пункт меню), пользователь кликает реальный UI
4. Шаг 3: оверлей со spotlight на ☰. Клик на ☰ → вызов `openDrawer()` через OnboardingContext → шаг 4
5. Шаг 4: оверлей со spotlight на «Данные» в открытом drawer. Клик → `closeDrawer()` + navigate('/data') → шаг 5
6. Шаг 5: тултип на странице /data (targetRect = null, без spotlight)
7. Шаги 6-7: карточки с кнопками (добавить/удалить тестовый актив)
8. При сбросе с данными в БД: шаги 6-7 пропускаются, после шага 5 сразу done

Монтируется в App.tsx поверх Routes, портал в document.body.

### PageTip

Переиспользуемый компонент. Принимает:
- `storageKey: string` — ключ localStorage
- `targetRef: RefObject<HTMLElement>` — элемент для подсветки
- `text: string` — текст подсказки

Логика: проверяет localStorage, если ключа нет — показывает CoachOverlay + CoachTooltip. По тапу — скрывает, записывает ключ.

Использование в страницах:
```tsx
// payments-page.tsx
const listRef = useRef<HTMLDivElement>(null);
<PageTip storageKey="hi-tip-payments" targetRef={listRef} text="История выплат..." />
```

### Состояние (localStorage)

| Ключ | Назначение |
|------|------------|
| `hi-onboarding-done` | Фаза 1 завершена |
| `hi-tip-payments` | Подсказка на /payments показана |
| `hi-tip-category` | Подсказка на /category показана |
| `hi-tip-asset` | Подсказка на /asset показана |

## Мокап главного экрана

Статичный React-компонент (MockMainPage.tsx) — копия главной страницы с примерными данными для шага 1.

**Содержимое мокапа:**
- Доход: ~42 380 ₽/мес (расчётный пассивный доход)
- Доходность: 8.2% · Портфель: 516 400 ₽
- Категории: Акции (3 шт, 31 200 ₽), Облигации (5 шт, 11 180 ₽)

**Требование к поддержке:** при изменении дизайна main page — обязательно обновлять мокап.

## Не затрагивается

- Splash screen (существующий, `hi-splash-seen`) — работает как раньше, онбординг запускается после него
- Существующее пустое состояние на главной («Пока нет активов...») — остаётся, видно на шаге 2
- Логика MOEX sync — используется как есть для синхронизации тестового актива
- Роутинг — без изменений
- Существующие компоненты AddAssetSheet/AddAccountSheet — тестовый актив добавляется программно, не через UI формы

## Зависимости

- Floating UI (`@floating-ui/react-dom` или `@floating-ui/react`) — для позиционирования тултипов. `@floating-ui/react-dom` уже в зависимостях через Radix. Проверить достаточность при реализации.
- Никаких новых внешних зависимостей (если `@floating-ui/react-dom` достаточен).
