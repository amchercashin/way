# Passive Income Tracker — Design Spec

## 1. Концепция и ценностное предложение

**Рабочее название:** CashFlow Tracker

**Суть:** Трекер пассивного дохода, который показывает расчётный денежный поток (₽/мес или ₽/год) со всех инвестиционных активов. Фокус на реальных деньгах, которые генерирует портфель, а не на бумажной переоценке.

**Целевая аудитория:** Российские частные инвесторы с диверсифицированным портфелем (акции, облигации, фонды, недвижимость, вклады).

**Ключевое отличие от конкурентов (Snowball Income, Portfolio Performance):**

- Главная метрика — нормализованный пассивный доход, а не стоимость портфеля
- Поддержка нефинансовых активов (недвижимость, вклады) наравне с биржевыми
- Local-first: данные на устройстве, без обязательной регистрации
- PWA: работает на любом телефоне без установки из App Store
- AI-импорт: пользователь может использовать любой AI для конвертации отчёта любого брокера

**Формат продукта:** Публичный продукт, local-first с опциональной облачной синхронизацией в будущем.

## 2. UI/UX Дизайн

### 2.1 Навигация и структура

Паттерн: карточный скролл (как Revolut/N26). Минимум навигации, progressive disclosure.

```
Главный экран (скролл)
├── Расчётный доход (мес ↔ год toggle)
├── Карточки категорий → [тап] → Экран категории
└── График истории выплат + CAGR (внизу)

Экран категории (напр. «Акции»)
├── Stat-блоки: Доход/мес, Стоимость, Доходность, Доля портф.
├── Список активов с доходом и частотой
└── График + CAGR (внизу)

Экран актива (напр. «SBER»)
├── Stat-блоки (те же 4)
├── Поля с тегами источника [MOEX] [импорт] [ручной]
├── Ожидаемый дивиденд (отсечка, выплата, зачисление)
└── График + CAGR (внизу)

☰ Drawer-меню
├── Мой доход (главный экран)
├── Календарь выплат
├── Импорт данных
├── Настройки
├── Экспорт/Бэкап
└── О приложении
```

### 2.2 Принципы дизайна

1. **Одно число — один взгляд.** Главный экран передаёт суть за 1 секунду. Расчётный доход — самый крупный элемент.
2. **Консистентность.** На всех трёх уровнях (Главный → Категория → Актив) одинаковая четвёрка stat-блоков: Доход/мес, Стоимость, Доходность, Доля портфеля.
3. **Визуальная симметрия.** Одинаковый столбчатый график с CAGR по центру на каждом уровне. Единый порядок элементов: Заголовок → Stat-блоки → Контент → Прогноз (если есть) → График.
4. **Прозрачность данных.** Каждое значение помечено тегом источника: [MOEX], [импорт], [ручной]. Пользователь всегда знает откуда цифра и может её переопределить.
5. **Progressive disclosure.** Главная → Категория → Актив → Детали. Каждый уровень добавляет глубину.
6. **Thumb zone.** Все интерактивные элементы в нижних 2/3 экрана. Mobile-first.
7. **Dark theme.** Основная цветовая схема — тёмная. Акцентный цвет: #4ecca3 (зелёный) для дохода.

### 2.3 Экраны

#### Главный экран

- Toggle мес/год (верхний правый)
- Расчётный пассивный доход — крупная зелёная цифра
- Подстрока: доходность % · стоимость портфеля
- Карточки категорий (цветной dot, название, кол-во позиций, доход, % от общего, chevron)
- График истории выплат с CAGR по центру (внизу)

#### Экран категории

- Stat-блоки: Доход/мес, Стоимость, Доходность, Доля портфеля
- Список активов: тикер, название, доход/мес, кол-во, стоимость, частота выплат (badge)
- Кнопка «+ Добавить»
- График + CAGR (внизу)

#### Экран актива

- Stat-блоки (те же 4)
- Поля данных с тегами источника:
  - Количество [импорт]
  - Последний дивиденд/купон [MOEX]
  - Частота выплат [MOEX]
- Блок «Ожидаемый дивиденд»:
  - Размер (прогноз): сумма на акцию × количество
  - Отсечка (ожид.): дата
  - Выплата (ожид.): дата
  - Зачисление (ожид.): дата
  - Источник: MOEX если объявлено, иначе экстраполяция прошлой выплаты
- Кнопка ✏️ → inline editing: тап на поле открывает редактирование прямо в карточке (не модал, не отдельный экран). Сохранение по blur/enter. Все поля редактируемы кроме расчётных (доход/мес, доходность, доля — они пересчитываются автоматически).
- График истории выплат + CAGR (внизу)

#### Экран импорта

- Toggle режима: ⟳ Обновить (полная синхронизация) / + Добавить (только новые сделки)
- Пояснение семантики каждого режима
- Способы импорта:
  1. Отчёт брокера (Сбер HTML/XLS — авто-парсинг)
  2. Через AI-помощник (промт для копирования → вставка MD-таблицы)
  3. CSV / Markdown файл
  4. Обновить с MOEX (цены, дивиденды, купоны)
  5. Добавить вручную (форма)

#### AI-импорт

- Готовый промт для копирования (кнопка 📋)
- Промт просит AI преобразовать отчёт брокера в Markdown-таблицу с колонками: Тикер, Название, Тип, Кол-во, Ср.цена, Посл.выплата, Частота
- Поле для вставки результата
- Кнопка «Распознать и импортировать»
- Формат: Markdown (человекочитаемый, легко проверить глазами, AI отлично генерирует)

#### Предпросмотр импорта

- Показывает diff: что обновлено, что новое, что без изменений
- Цветовая кодировка: зелёный = обновлено, жёлтый = новое, серый = без изменений
- Сводка: N изменено, N новых, N без изменений
- Конфликты: если импортируемое значение конфликтует с ручным переопределением, показываем оба значения и даём пользователю выбрать
- Кнопки: Отмена / Применить

#### Календарь выплат

- Хронологический список ожидаемых поступлений по месяцам
- Каждая запись: тикер/название, тип выплаты, сумма, дата
- Показывает ближайшие 3 месяца по умолчанию, с кнопкой «Показать ещё» для загрузки следующих

#### Drawer-меню

- Навигация по секциям: Мой доход, Календарь выплат, Импорт данных, Настройки, Экспорт/Бэкап, О приложении

#### Настройки

- Период по умолчанию (мес/год)
- Валюта отображения (₽ для MVP)
- Автообновление данных MOEX (вкл/выкл)
- Управление данными: очистить все данные, сбросить приложение
- Показать онбординг заново

### 2.4 Ключевые сценарии

**Первый запуск:**
1. Приветствие — объяснение сути (1 экран)
2. «Как добавить активы?» → выбор способа импорта
3. Добавление первого актива
4. Автоматическая подгрузка данных с MOEX (для биржевых)
5. Главный экран с первым расчётным доходом

**Еженедельная докупка:**
1. ☰ → Импорт → Загрузить отчёт Сбера (или AI-импорт)
2. Предпросмотр: «Найдено N новых сделок»
3. Подтвердить → Позиции обновлены, доход пересчитан

**Проверка и корректировка:**
1. Тап на категорию → Тап на актив
2. Видит теги источников
3. ✏️ → Редактирование поля → Тег сменяется на [ручной]

## 3. Архитектура и стек

### 3.1 Технический стек

| Компонент | Технология |
|-----------|-----------|
| Тип | PWA (Progressive Web App), client-side only |
| Фреймворк | React 19 + TypeScript |
| Сборка | Vite |
| UI | Tailwind CSS + shadcn/ui (dark theme) |
| Хранение | IndexedDB через Dexie.js (local-first) |
| Офлайн | Service Worker (Workbox) |
| Парсинг XLS/HTML | SheetJS (xlsx) + собственный маппер для Сбера |
| Парсинг Markdown | Собственный парсер MD-таблиц |
| Данные о выплатах | MOEX ISS API (бесплатно, без авторизации) |
| Деплой | Vercel или Netlify (free tier) |
| Бэкенд | Нет (zero backend для MVP) |

### 3.2 Почему PWA, а не нативное приложение

- $0 на инфраструктуру, мгновенный деплой
- Устанавливается на телефон (Add to Home Screen), полноэкранный режим
- Работает офлайн с Service Worker
- Нет модерации App Store
- React-код переиспользуется в React Native на 60-80% при переходе на натив
- Быстрая итерация: от идеи до MVP за недели

### 3.3 MOEX ISS API

- Эндпоинт дивидендов: `https://iss.moex.com/iss/cci/corp-actions/dividends`
- Эндпоинт купонов: `https://iss.moex.com/iss/cci/corp-actions/coupons`
- CORS: API отдаёт `Access-Control-Allow-Origin` для любого origin — прокси не нужен
- Форматы ответа: JSON, XML, CSV
- Аутентификация: не требуется
- Ограничения: 15-минутная задержка на рыночные данные (для нас не критично)
- Текущие цены: `/iss/engines/stock/markets/shares/securities/{ticker}.json` (акции), `/iss/engines/stock/markets/bonds/securities/{ticker}.json` (облигации)

### 3.4 Парсер отчёта Сбера

- Сбер отправляет отчёты в формате HTML (по email) и XLS (из приложения/ЛК)
- Парсинг: SheetJS для XLS, DOM parser для HTML
- Логику маппинга колонок пишем сами, ориентируясь на open source проект [Investbook](https://github.com/spacious-team/investbook) (AGPL-3.0, парсит отчёты Сбера)
- Извлекаемые данные: позиции (тикер, количество, средняя цена), сделки, дивиденды, купоны
- Повторная загрузка = обновление без дублей

## 4. Модель данных

### 4.1 Ключевые сущности

```typescript
type DataSource = 'moex' | 'import' | 'manual';

interface Asset {
  id: string;
  type: AssetType;
  ticker?: string;        // для биржевых
  name: string;
  quantity: number;
  averagePrice?: number;  // для облигаций: в рублях (номинал × цена%)
  currentPrice?: number;  // для облигаций: в рублях (номинал × цена%)
  faceValue?: number;     // номинал облигации в рублях (напр. 1000)
  dataSource: DataSource;
  createdAt: Date;
  updatedAt: Date;
}

type AssetType = 'stock' | 'bond' | 'fund' | 'realestate' | 'deposit' | 'other';

interface PaymentSchedule {
  id: string;
  assetId: string;
  frequencyPerYear: number;  // 1, 2, 4, 12
  /**
   * Размер выплаты ЗА ЕДИНИЦУ актива:
   * - Акции: дивиденд на 1 акцию (₽)
   * - Облигации: купон на 1 облигацию (₽)
   * - Недвижимость: полная сумма аренды (quantity = 1)
   * - Вклады: сумма процентов за период (quantity = 1)
   * - Прочее: сумма за период (quantity = 1)
   *
   * Для нефинансовых активов quantity всегда = 1,
   * а lastPaymentAmount = полная сумма дохода за период.
   */
  lastPaymentAmount: number;
  lastPaymentDate?: Date;
  nextExpectedDate?: Date;
  nextExpectedCutoffDate?: Date;
  nextExpectedCreditDate?: Date;
  dataSource: DataSource;
}

interface PaymentHistory {
  id: string;
  assetId: string;
  amount: number;          // полная сумма выплаты (уже × quantity)
  date: Date;
  type: 'dividend' | 'coupon' | 'rent' | 'interest' | 'distribution' | 'other';
  dataSource: DataSource;
}

interface ImportRecord {
  id: string;
  date: Date;
  source: 'sber_xls' | 'sber_html' | 'csv' | 'markdown' | 'ai_import' | 'manual';
  mode: 'update' | 'add';
  itemsChanged: number;
  itemsAdded: number;
  itemsUnchanged: number;
}
```

### 4.2 Расчёт нормализованного дохода

```
Расчётный доход/мес = Σ (lastPaymentAmount × quantity × frequencyPerYear) / 12

Расчётный доход/год = Σ (lastPaymentAmount × quantity × frequencyPerYear)

Доходность = (Расчётный доход/год) / (Σ currentPrice × quantity) × 100%

CAGR = ((income_last_year / income_first_year) ^ (1 / N_years)) - 1
  где income_year = суммарный фактический пассивный доход за календарный год
  (из PaymentHistory). Показывается при наличии 2+ полных годовых периодов.
```

**Допущения MVP:** Расчётный доход использует последнюю известную выплату (`lastPaymentAmount`), экстраполированную равномерно на весь год. Это приближение — реальные выплаты могут отличаться (нерегулярные купоны, переменные дивиденды). Пользователь может скорректировать вручную.

Каждое значение помечено `DataSource`. При ручном переопределении поля `dataSource` всей сущности меняется на `'manual'`. Это означает потерю гранулярности: если пользователь изменил только `currentPrice`, а `quantity` пришло из импорта, оба будут помечены как manual. Это осознанный trade-off MVP — per-field source tracking добавим при необходимости.

**CAGR: определение полного года.** Полный год = календарный год (янв-дек). Первый и последний неполные годы исключаются из расчёта. Если пользователь начал в июне 2025, первый полный год = 2026.

### 4.3 Облигации: особенности расчёта

Цены облигаций на MOEX котируются в % от номинала. В модели данных хранятся в рублях:
- `currentPrice` = `faceValue × (котировка% / 100)` — рыночная стоимость в рублях
- `averagePrice` — средняя цена покупки в рублях
- `lastPaymentAmount` — купон на 1 облигацию в рублях (абсолютная величина)

Конвертация из % в рубли происходит на этапе импорта/обновления с MOEX.

### 4.4 Хранение

IndexedDB через Dexie.js. Таблицы:
- `assets` — активы
- `paymentSchedules` — расписания выплат
- `paymentHistory` — история выплат
- `importRecords` — записи импортов
- `settings` — настройки приложения

Экспорт/бэкап: JSON-файл со всеми таблицами. Импорт бэкапа = полное восстановление.

Миграция схемы: Dexie.js поддерживает версионирование схемы — при обновлении приложения IndexedDB мигрирует автоматически.

### 4.5 Обработка ошибок MOEX API

- При сетевых ошибках: показываем кэшированные данные с меткой «обновлено: [дата]»
- При недоступности API: приложение полностью работоспособно на локальных данных
- Пользователь может запустить обновление вручную через ⟳ на главном экране

### 4.6 Валидация при редактировании

- Количество: целое положительное число (> 0)
- Цены и суммы выплат: положительное число (> 0)
- Частота: выбор из списка (1, 2, 4, 12)
- Даты: не в будущем для фактических выплат; любая дата для ожидаемых

## 7. MVP Scope

### Включено

1. Главный экран с расчётным пассивным доходом (мес/год toggle)
2. Drill-down: категория → актив → детали с редактированием
3. 6 типов активов (акции, облигации, фонды, недвижимость, вклады, прочее)
4. Импорт: ручной ввод + отчёт Сбера (HTML/XLS) + CSV/MD + AI-помощник с промтом
5. Режимы импорта: обновить / добавить с предпросмотром diff
6. MOEX ISS API: цены, дивиденды, купоны для биржевых бумаг
7. Нормализация выплат к ₽/мес и ₽/год
8. График истории выплат с CAGR на всех уровнях
9. Ожидаемый дивиденд/купон с датами (отсечка, выплата, зачисление)
10. Календарь выплат
11. Drawer-меню
12. Экспорт/бэкап (JSON)
13. Онбординг (первый запуск)

### Вне MVP

- API брокеров (Т-Инвестиции, Finam, Alor)
- Облачная синхронизация / бэкап в облако
- Прогнозирование дивидендов (dohod.ru)
- Push-уведомления
- React Native (нативное приложение)
- Portfolio Open Format (JSON-стандарт)
- Парсеры для других брокеров (ВТБ, Т-Банк и др.)
- Multi-currency поддержка
- Налоговый учёт
